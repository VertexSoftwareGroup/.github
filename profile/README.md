# Welcome to Vertex Software Group 🚀

Welcome to the official code hub for **Vertex Software Group**. We are a development collective dedicated to building, maintaining, and preserving private game server infrastructure, custom tooling, and community platforms. 

Our mission is two-fold: providing high-performance, stable environments for our **actively running live services**, while acting as a digital museum to **preserve the history** of our discontinued (End-of-Support) projects.

---

## 🗺️ Navigation & Roadmap

To keep our workspace clean, our repositories are strictly categorized into two states: **Active Ecosystems** and **The Heritage Vault**. 

### 🟢 1. Active Ecosystems (Live Services)
These repositories contain the source code, configurations, and tools for servers that are currently online and actively maintained. 
* *Contributions, bug reports, and pull requests are welcome here.*
* **Look for these tags:** `#live`, `#active`, `#production`

### 📦 2. The Heritage Vault (End-of-Support / EoS)
When a server or tool reaches its End-of-Support phase, we do not delete it. Instead, we permanently archive it here. 
* *These repositories are set to **Read-Only**. They serve as a learning resource, a backup for the community, and a historical record of our development journey.*
* **Look for these tags:** `#eos`, `#discontinued`, `#legacy`, `#archive`

## 📜 Community Guidelines & Open Source

We believe in open-source gaming development, but maintaining a private server ecosystem requires rules to keep our community safe and fair.

1. **Educational Purposes:** Unless stated otherwise in a repository's specific LICENSE file, our emulators and tools are published for educational, archival, and research purposes.
2. **Security Vulnerabilities:** If you discover a critical exploit or security flaw in any of our *active* servers, please do not open a public issue. Instead, report it privately to our core administration team.
3. **Be Respectful:** Treat fellow developers and community members with respect in issues and discussions. Code reviews should be constructive, not destructive.
